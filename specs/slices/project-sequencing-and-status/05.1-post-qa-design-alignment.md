# Slice 05.1 — Post-QA alignment, agent surfacing, numbered folders

**Goal:** Three pipeline improvements: (1) number project subfolders in pipeline order so they sort naturally; (2) surface the proposed agent team at tasks_ready before `/implement` runs; (3) after QA passes, review the design doc and future slices for alignment issues, summarize and apply any changes, leave them uncommitted.

## Happy path

- Project folder structure uses numbered subfolders in pipeline order: `01-design/`, `02-slices/`, `03-briefs/`, `04-tasks/`, `05-qa/`. All command files (design.md, slice.md, spec.md, implement.md, qa.md) updated to use new paths.
- Tasks created → tasks_ready gate output includes proposed agent team (unique `agent_type` values from tasks), explicitly labels it as single agent or team, asks for confirmation before the human runs `/implement`
- QA passes → agent reviews design doc and all `draft`/`reviewed` slices for alignment issues → summarizes changes (or "no updates needed") → applies any changes → leaves uncommitted alongside implementation diff → advances to `signoff_review`

## Edge cases

- Multiple future slices need updating: update all, list each change in the signoff summary
- Design doc update conflicts with a future slice: resolve both together, note the relationship in the summary
- No future slices exist (last slice): skip slice review, check design doc only
- Human wants to add an agent type not in any task: update task `agent_type` fields directly, then re-confirm
